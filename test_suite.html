MS-01 · Geolocalización + Mapa Leaflet
navigator.geolocation + OpenStreetMap tiles
Quién: QA / Dev
Cuándo: Pre-deploy
Dónde: Browser
# Nombre de Prueba Tipo Qué Verifica Resultado Esperado Estado
GEO-U01 API Disponible unitaria caja blanca navigator.geolocation existe en window truthy, no undefined skip
GEO-U02 getCurrentPosition Callback unitaria caja blanca La función llama successCallback con coords lat ∈ [-90,90], lng ∈ [-180,180] skip
GEO-U03 Error Permiso Denegado unitaria caja negra errorCallback recibe code=1 (PERMISSION_DENIED) UI muestra mensaje de error skip
GEO-I01 Integración Leaflet integración caja negra Coords reales colocan marcador en mapa Marker visible en posición correcta skip
GEO-I02 Tiles OpenStreetMap integración caja negra Tiles cargan desde tile.openstreetmap.org HTTP 200, imágenes visibles skip
GEO-UI01 Mapa Renderizado en DOM ui caja negra Elemento #map tiene dimensiones > 0 width > 0 && height > 0 skip
GEO-UI02 Coordenadas en Pantalla ui caja blanca Lat/Lng mostradas en elemento visible Formato: ##.#####, -##.##### skip
// Criterios de Aceptación · MS-01 · Geo
  • El navegador debe tener soporte para navigator.geolocation (cobertura >97% navegadores modernos)
  • Latitud dentro de [-90, 90] y longitud dentro de [-180, 180]
  • Timeout máximo de 10 000 ms antes de mostrar error al usuario
  • Si el usuario deniega permiso, se muestra mensaje informativo (no falla silenciosa)
  • Mapa Leaflet inicializado con zoom ≥ 13 y marcador centrado en posición real
  • Tiles cargan completamente en ≤ 3 s con conexión normal

// Leyenda de tipos
unitaria Prueba un solo módulo/función aislado
integración Verifica la comunicación entre componentes/APIs
ui Verifica que la pantalla responde correctamente
caja blanca Conoce la lógica interna (verifica el código)
caja negra Solo entradas/salidas (ignora internos)
